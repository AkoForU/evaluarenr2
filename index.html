Tutorial: Cum să clonezi, să gestionezi și să colaborezi cu Git

În acest tutorial, vei învăța cum să clonezi un repository Git de pe GitHub, să faci modificări, să creezi branch-uri, să faci pull request-uri și să colaborezi. Vom folosi un exemplu real: clonarea repository-ului "evaluarennr2".

Pasul 1: Deschide terminalul

Navighează în folderul dorit folosind comanda cd. De exemplu, dacă vrei să clonezi pe Desktop:

cd Desktop

Pasul 2: Clonează repository-ul

Folosește comanda git clone urmată de URL-ul repository-ului. În exemplul nostru:

git clone https://github.com/AkoForU/evaluarennr2.git

Pasul 3: Verifică rezultatul clonării

După ce rulezi comanda, vei vedea un output similar cu cel de mai jos, care indică progresul clonării:

Cloning into 'evaluarennr2'... remote: Enumerating objects: 3, done. remote: Counting objects: 100% (3/3), done. remote: Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) Receiving objects: 100% (3/3), done.

Pasul 4: Navighează în folderul clonat

Intră în folderul repository-ului clonat folosind comanda cd:

cd evaluarennr2

Pasul 5: Verifică starea repository-ului

Folosește comanda git status pentru a verifica starea curentă a repository-ului:

git status

În exemplul nostru, output-ul arată că există un fișier neurmărit (settings.xml):

On branch main Your branch is up to date with 'origin/main'. Untracked files: (use "git add ..." to include in what will be committed) settings.xml nothing added to commit but untracked files present (use "git add" to track)

Pasul 6: Adaugă fișierul pentru commit

Folosește comanda git add pentru a adăuga fișierul settings.xml în staging area:

git add .

Notă: git add . adaugă toate fișierele modificate sau neurmărite. Dacă vrei să adaugi doar un fișier specific, folosește git add settings.xml.

Pasul 7: Creează un commit

Folosește comanda git commit pentru a salva modificările cu un mesaj descriptiv:

git commit -m "Commit file settings.xml"

Output-ul va arăta astfel:

[main 5f7e5c9] Commit file settings.xml 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 settings.xml

Pasul 8: Verifică starea după commit

Folosește din nou git status pentru a verifica starea repository-ului:

git status

Output-ul va arăta că ai un commit care nu a fost încă trimis pe GitHub:

On branch main Your branch is ahead of 'origin/main' by 1 commit. (use "git push" to publish your local commits) nothing to commit, working tree clean

Pasul 9: Verifică starea după modificări

După ce faci modificări (de exemplu, redenumești settings.xml în configuration.xml și adaugi un fișier nou configuration.xml), rulează din nou git status:

git status

Output-ul va arăta modificările și fișierele noi:

On branch main Your branch is ahead of 'origin/main' by 1 commit. (use "git push" to publish your local commits) Changes not staged for commit: (use "git add/rm ..." to update what will be committed) (use "git restore ..." to discard changes in working directory) deleted: settings.xml Untracked files: (use "git add ..." to include in what will be committed) configuration.xml no changes added to commit (use "git add" and/or "git commit -a")

Pasul 10: Adaugă modificările pentru commit

Folosește git add . pentru a adăuga toate modificările (inclusiv fișierul redenumit și cel nou):

git add .

Pasul 11: Creează un nou commit

Folosește git commit pentru a salva modificările:

git commit -m "Commit file configuration.xml"

Output-ul va arăta astfel:

[main 08f77f7e] Commit file configuration.xml 1 file changed, 0 insertions(+), 0 deletions(-) rename settings.xml => configuration.xml (100%)

Pasul 12: Trimite modificările pe GitHub

Folosește git push pentru a trimite commit-urile pe GitHub:

git push

Output-ul va arăta progresul:

Enumerating objects: 6, done. Counting objects: 100% (6/6), done. Delta compression using up to 12 threads Compressing objects: 100% (4/4), done. Writing objects: 100% (5/5), 535 bytes | 535.00 KiB/s, done. Total 5 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) To https://github.com/AkoForU/evaluarennr2.git e66c0c4..08f77f7e main -> main

Pasul 13: Creează un fișier nou

Creează un fișier nou numit page1.html folosind comanda touch:

touch page1.html

Pasul 14: Adaugă și comite fișierul nou

Adaugă fișierul page1.html în staging area și comite-l:

git add .
git commit -m "Commit page1.html"

Output-ul va arăta astfel:

[main fbdd22a] Commit page1.html 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 page1.html

Pasul 15: Trimite fișierul pe GitHub

Folosește git push pentru a trimite noul commit pe GitHub:

git push

Output-ul va arăta progresul:

Enumerating objects: 3, done. Counting objects: 100% (3/3), done. Delta compression using up to 12 threads Compressing objects: 100% (2/2), done. Writing objects: 100% (2/2), 287 bytes | 287.00 KiB/s, done. Total 2 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) To https://github.com/AkoForU/evaluarennr2.git 08f77f7e..fbdd22a main -> main

Pasul 16: Sincronizează cu repository-ul remote

Folosește git pull pentru a te asigura că ești la zi cu repository-ul remote:

git pull

Dacă ești deja la zi, output-ul va arăta:

Already up to date.

Pasul 17: Revino la un commit anterior

Folosește git reset --hard HEAD~1 pentru a reveni la commit-ul anterior:

git reset --hard HEAD~1

Output-ul va arăta:

HEAD is now at 08f77f7e Commit file configuration.xml

Pasul 18: Trimite modificările forțat

După resetare, folosește git push --force pentru a actualiza repository-ul remote:

git push --force

Output-ul va arăta:

Total 0 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) To https://github.com/AkoForU/evaluarennr2.git + fbdd22a...08f77f7e main -> main (forced update)

Pasul 19: Creează un branch nou

Creează un branch nou numit stage și comută pe el:

git checkout -b stage

Output-ul va arăta:

Switched to a new branch 'stage'

Pasul 20: Creează și comite un fișier nou pe branch-ul stage

Creează un fișier nou hosts.xml, adaugă-l și comite-l:

touch hosts.xml
git add .
git commit -m "Commit file hosts.xml"

Output-ul va arăta:

[stage e8baed2] Commit file hosts.xml 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 hosts.xml

Pasul 21: Trimite branch-ul stage pe GitHub

Folosește git push origin stage pentru a trimite branch-ul pe GitHub:

git push origin stage

Output-ul va arăta:

Enumerating objects: 3, done. Counting objects: 100% (3/3), done. Delta compression using up to 12 threads Compressing objects: 100% (2/2), done. Writing objects: 100% (2/2), 288 bytes | 288.00 KiB/s, done. Total 2 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) remote: remote: Create a pull request for 'stage' on GitHub by visiting: remote: https://github.com/AkoForU/evaluarennr2/pull/new/stage remote: To https://github.com/AkoForU/evaluarennr2.git * [new branch] stage -> stage

Pasul 22: Comută pe branch-ul main și sincronizează

Comută pe branch-ul main și sincronizează cu remote:

git checkout main
git pull origin main

Output-ul va arăta:

Switched to branch 'main' Your branch is up to date with 'origin/main'. From https://github.com/AkoForU/evaluarennr2 * branch main -> FETCH_HEAD Already up to date.

Pasul 23: Mergi modificările din branch-ul stage

Mergi modificările din branch-ul stage în main:

git merge stage

Output-ul va arăta:

Updating 08f77f7e..e8baed2 Fast-forward hosts.xml | 0 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 hosts.xml

Pasul 24: Trimite modificările pe branch-ul main

Folosește git push pentru a trimite modificările pe main:

git push

Output-ul va arăta:

Total 0 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0) To https://github.com/AkoForU/evaluarennr2.git 08f77f7e..e8baed2 main -> main

Pasul 25: Creează un branch nou dev

Creează un branch nou numit dev și comută pe el:

git checkout -b dev

Output-ul va arăta:

Switched to a new branch 'dev'

Pasul 26: Creează și comite un fișier nou pe branch-ul dev

touch env.txt
git add .

Output-ul va arăta:

[dev 2e01115] Committed file env.txt 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 env.txt

Pasul 27: Comută pe branch-ul stage

Comută pe branch-ul stage pentru a face modificări suplimentare:

git checkout stage

Output-ul va arăta:

Switched to branch 'stage'

Pasul 28: Mergi modificările din branch-ul dev în stage

Mergi modificările din branch-ul dev în stage:

git merge dev

Output-ul va arăta:

Updating e8baed2..2e01115 Fast-forward env.txt | 0 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 env.txt

Pasul 29: Trimite modificările pe branch-ul stage

Folosește git push origin stage pentru a trimite modificările pe branch-ul stage pe GitHub:

git push origin stage

Output-ul va arăta:

Enumerating objects: 3, done. Counting objects: 100% (3/3), done. Delta compression using up to 12 threads Compressing objects: 100% (2/2), done. Writing objects: 100% (2/2), 233 bytes | 233.00 KiB/s, done. Total 2 (delta 1), reused 0 (delta 0), pack-reused 0 (from 0) remote: Resolving deltas: 100% (1/1), completed with 1 local object. To https://github.com/AkoForU/evaluarennr2.git e8baed2..2e01115 stage -> stage

Pasul 30: Creează un pull request pentru branch-ul stage

Output-ul de la git push origin stage îți oferă un link pentru a crea un pull request. Accesează link-ul furnizat pentru a crea un pull request pe GitHub:

remote: Create a pull request for 'stage' on GitHub by visiting: remote: https://github.com/AkoForU/evaluarennr2/pull/new/stage

Comută pe branch-ul main și sincronizează cu remote:
